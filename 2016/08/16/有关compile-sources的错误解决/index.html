有关Compile sources的错误解决

2016-08-16 | iOS

今天遇到两个小问题，都是编译错误，花了点时间

一、Xcode error

  1. no input files，文件路径报错；

  2. duplicate symbols for architecture x86_64，重复引用，编译失败；

二、解决方法

clang: error: no input files 这个问题一般都是因为删除或者移动了某一个文件，但是在你的编译资源里面还存在它，只要在（project > target > Build Phases > Compile Sources）里面把那些白色的文件（与其他文件样式上有变化）删除掉，就可以编译通过了。

duplicate symbols for architecture x86_64 重复引用问题一般有两个原因：

  1. 头文件在工程中引用了多次；
  2. 在（project > target > Build Phases > Compile Sources）里面有相同的头文件存在，把那些白色的文件（与其他文件样式上有变化）删除掉，就可以编译通过了。
