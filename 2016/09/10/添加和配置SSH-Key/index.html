添加和配置SSH Key

2016-09-10 | iOS

GitLab是一个自托管的Git项目仓库，可以通过Web页面访问公开或者私人的项目。它拥有与Github类似的功能，能够浏览源代码，管理团队的访问权限，非常容易就可以查看历史记录。对于个人来说，工作中写的一些Demo、封装的控件都可以上传到Gitlab上，既安全又不会丢失，也便于更新和维护。而SSH key呢，则提供了一种与Gitlab的通信方式。通过这种方式，能够在不用输入密码的情况下（输密码也是件头疼的事呢），就可以进行版本控制。

步骤：

  1. 检查SSH Key是否存在，如果存在直接跳到第三步；
  2. 如果不存在，就创建生成新的SSH Key；
  3. 将SSH Key添加到Gitlab中；

一、检查SSH Key是否存在

输入以下命令，如果id_rsa.pub 或 id_dsa.pub文件存在，则直接进入步骤三：将SSH key添加到GitHub中，否则会提示：NO such file or directory，则进入第二步生成SSH Key。

二、生成新的SSH Key

2.1 生成public/private rsa key pair文件

输入 ssh-keygen -t rsa -C “your_email@example.com“ 命令,生成 public/private rsa key pair 文件。默认会在相应路径下（/your_home_path）生成id_rsa和id_rsa.pub两个文件，成功以后会有 Generating public/private rsa key pair.Enter file in which to save the key的提示。
